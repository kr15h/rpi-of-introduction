Introduction to openFrameworks

on the Raspberry Pi

Check connection to the Pi

Make sure that you are able to connect to the Raspberry Pi by issuing this in the terminal:

ssh pi@your.pi.ip.address

Replace your.pi.ip.address with the actual ip address of your Pi (eg. 192.168.2.200)

Setup remote desktop

get XQuartz for Mac

get MobaXterm for Windows

nothing extra for Linux

Access Pi remote desktop

In Mac and Linux Terminal just type:

ssh -X pi@your.pi.ip.addr

Replace your.pi.ip.addr with the actual IP address of the Pi.

Connect using MobaXterm on Windows. Remote desktop capability should be enabled by default there.

openFrameworks overview

As said in the openFrameworks website:

openFrameworks is an open source C++ toolkit for creative coding.


True it is...

Based on C++

openFrameworks is written in C++ and basically it is C++ with all it's pros and cons.

Project directory structure

There are two main directories:

  • src
    for the source .h and .cpp files
  • bin
    for the compiled program files

File types

There are two basic file types:

  • .h
    or header files
  • .cpp
    or C++ files

Main functions

openFrameworks has three main functions:

  • setup()
    Does the initial setup when the program starts
  • update()
    Updates data, this is where to do the math
  • draw()
    Draw images, videos or anything else

Makefile

There has to be a Makefile in the project folder to be able to compile the project. If it is not there, copy it from some of the default projects like emptyExample:

cp ../emptyExample/Makefile ./

Using addons

You need to have a addons.make file in your project directory. Simply enter the folder name of the addon you want to use in the file, separate multiple addons with a newline:

ofxMidi
ofxOsc

Sum-up

The process is basically the same for almost all projects:

  1. Edit code
  2. Upload to the Raspberry Pi
  3. Compile and run
